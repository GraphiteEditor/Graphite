Is there a way we could separate the TaggedValue's type (the enum discriminant part) from its value (the enum value part) using type erasure? Then the value part could be something like Option<Box<std::any::Any>>, making it optional. So we set it if it's provided by a widget, but otherwise we omit it if it's not controlled by a widget, and thus avoid needing a way to even serialize it if it's never given a widget (like because it's a primary input which is never given a widget).
